I am workign in Python and Streamlit, working with a NASA JSON file. I already have some code filled out for a function I am trying to set up called:

but found out I could use an import from geopy called geodesic. could you help me complete the function using this

Response

from geopy.distance import geodesic

def filter_by_location(df, user_lat, user_lon, radius_miles):
    filtered = df.copy()
    distances = []

    for _, row in filtered.iterrows():
        station_coords = (row["lat"], row["lon"])
        user_coords = (user_lat, user_lon)
        distance = geodesic(user_coords, station_coords).miles
        distances.append(distance)

    filtered["distance_miles"] = distances
    filtered = filtered[filtered["distance_miles"] <= radius_miles]

    return filtered

A few things to note:
•	geodesic(point1, point2).miles takes two (lat, lon) tuples and returns the distance. It uses the WGS-84 ellipsoid model, which is more accurate than the haversine formula for real-world distances.
•	The distances are stored in a new column "distance_miles" on the copied dataframe, so your original df stays untouched.
•	The final filter keeps only rows where the station falls within your radius_miles.
•	Make sure your column names "lat" and "lon" match what's actually in your NASA JSON — they might be something like "latitude" / "longitude" depending on the dataset.
You may also want to sort by distance before returning, which is handy for displaying nearest stations first in Streamlit:

return filtered.sort_values("distance_miles").reset_index(drop=True)

Step 1: Start with the outer loop

Step 2: Add the condition — but the condition itself involves a loop
That inner loop with a break is asking: "does at least one category match?" That's exactly what any() does:
python
any(c["title"] in selected_titles for c in e.get("categories", []))
Step 3: Plug it in as the filter
python
events = [
    e for e in all_events
    if any(c["title"] in selected_titles for c in e.get("categories", []))
]
This is equivalent to your original because:
•	The for/break pattern stops as soon as it finds one match → any() short-circuits the same way
•	append(e) once per event → the list comprehension naturally includes e once
The selected_titles = set(selected_labels) line stays as-is — it's already efficient and doesn't need to change.
